# Portfolio Data Loading Guide

## For KITLabs Admin Team

This guide explains how to populate the **Portfolio** section in the Leads Portal with KITLabs services and project data. The Portfolio helps the sales team prepare for customer calls and meetings by providing pitch scripts, project references, and shareable resources.

---

## Getting Started

1. Log into the **Admin Portal** at `https://leadsportaladmin.kitlabs.us`
2. Click **Portfolio** in the left sidebar
3. You'll see two tabs: **Services** and **Projects**

---

## Step 1: Add Services

Services represent what KITLabs offers (e.g., Mobile App Development, Web Development, Cloud Solutions). Start by adding all the services from **kitlabs.us**.

### How to Add a Service

1. Click **"+ New Service"** on the Portfolio page
2. Fill in the form:

| Field | What to Enter | Where to Find It |
|-------|--------------|-----------------|
| **Service Name** | The service offering name | kitlabs.us services page |
| **Description** | Detailed description of what this service includes | kitlabs.us service detail pages |
| **Email Script** | A pitch email template that admin can copy and send to leads | Write based on the service — include key benefits, typical deliverables, and a call-to-action |
| **Phone Script** | Talking points for phone calls with prospects | Key selling points, common objections and responses, qualifying questions |
| **Meeting Script** | Presentation talking points for meetings | Opening, agenda, key slides to reference, closing and next steps |
| **URLs** | Links admin can share with customers | Add the kitlabs.us service page URL, any relevant blog posts, case studies |
| **Documents** | PowerPoint decks, PDFs, proposals | Add links to Google Slides/Drive pitch decks, brochures, or one-pagers |

3. Click **"Create Service"**

### Recommended Services to Add (from kitlabs.us)

- Mobile App Development (iOS & Android)
- Web Application Development
- UI/UX Design
- Cloud Solutions & Migration
- E-Commerce Development
- Custom Software Development
- API Development & Integration
- DevOps & CI/CD
- AI & Machine Learning Solutions
- Startup MVP Development

### Tips for Writing Pitch Scripts

**Email Script** — Keep it concise (3-4 paragraphs):
- Opening: Reference the customer's industry/need
- Value prop: What KITLabs brings to the table for this service
- Social proof: Mention a relevant project or client
- CTA: Suggest a meeting or demo

**Phone Script** — Use bullet points:
- Introduction (15 seconds)
- Discovery questions (ask about their current situation)
- Key benefits (3-4 points)
- Handling objections (common pushbacks)
- Close (book a meeting)

**Meeting Script** — Structure for a 30-min presentation:
- Intro & agenda (2 min)
- Understanding their needs (5 min)
- Our approach for this service (10 min)
- Relevant case studies (5 min)
- Timeline & pricing overview (5 min)
- Q&A and next steps (3 min)

## Step 2: Add Projects

Projects are completed work that showcases KITLabs capabilities. Each project links to a service and serves as social proof during sales conversations.

### How to Add a Project

1. Click **"+ New Project"** on the Portfolio page (or from within a Service detail page click "Add Project")
2. Fill in the form:

| Field | What to Enter | Example |
|-------|--------------|---------|
| **Title** | Project name | "FinTech Mobile Banking App" |
| **Description** | What was built and the business impact | "Built a full-stack mobile banking app with real-time transactions, biometric auth, and analytics dashboard" |
| **Service** | Select the related service from dropdown | "Mobile App Development" |
| **Category** | Industry category | "FinTech", "Healthcare", "E-Commerce", "Education" |
| **Domain** | Business domain | "Finance", "Retail", "Government" |
| **Technologies** | Tech stack used (type and press Enter for each) | "React Native", "Node.js", "PostgreSQL", "AWS" |
| **Client Name** | Customer name (if shareable) | "Acme Financial" |
| **Client Details** | Background on the client (optional) | "Mid-size credit union serving 50k members in Ontario" |
| **Demo Video URL** | Link to a demo recording | YouTube or Loom link |
| **Documents** | Pitch deck, case study PDF | Google Drive/Slides links |
| **Completed Date** | When the project was delivered | Select date |
| **Email Script** | Project-specific pitch for emails | "We recently delivered a similar project for a financial services company..." |
| **Phone Script** | Phone talking points referencing this project | Key results, timeline, tech decisions |
| **Meeting Script** | Meeting talking points with this as a case study | Problem → Solution → Results format |

3. Click **"Create Project"**

### What Information to Gather for Each Project

From **kitlabs.us portfolio** and internal knowledge:

1. **Project overview** — What was built, for whom, and why
2. **Technologies used** — Full tech stack (frontend, backend, database, cloud, tools)
3. **Key results** — Metrics if available (users, revenue impact, performance improvements)
4. **Timeline** — How long did it take to deliver
5. **Client info** — Who the client is (if shareable publicly)
6. **Demo** — Any recorded walkthrough or live demo link
7. **Pitch deck** — If a presentation was created for this project, add the link

## Step 3: Add Documents & URLs

### For Each Service, Add:
- **Service page URL** from kitlabs.us (e.g., `https://kitlabs.us/services/mobile-app-development`)
- **Pitch deck** in Google Slides (create one if it doesn't exist)
- **One-pager PDF** summarizing the service (if available)
- **Blog posts** related to the service
- **Case studies** relevant to the service

### For Each Project, Add:
- **Case study document** (if available)
- **Pitch deck** referencing this project
- **Demo video** (Loom or YouTube)
- **Technical architecture diagram** (if shareable)

## How Admins Use the Portfolio

### Before a Customer Call
1. Go to **Portfolio → Services** and find the relevant service
2. Read the **Phone Script** — copy key talking points
3. Check the linked **Projects** for relevant case studies
4. Open a project detail page for specific metrics and results

### Before Sending a Pitch Email
1. Go to the relevant **Service** or **Project** detail page
2. Click **"Copy"** on the **Email Script**
3. Paste into the email compose on the lead's detail page
4. Customize with the customer's name and specific needs

### Before a Meeting
1. Go to the relevant **Service** detail page
2. Read the **Meeting Script** for structure and talking points
3. Open the **Documents** section and grab the pitch deck link
4. Review linked **Projects** for case studies to reference
5. Share relevant **URLs** with the customer before/after the meeting

### Sharing Links with Customers
1. On any Service or Project page, find the **URLs** section
2. Click the **copy icon** next to any URL
3. Paste into an email or message to the customer

---

## Maintaining the Portfolio

- **Update regularly**: Add new projects as they're completed
- **Keep scripts current**: Update pitch scripts based on what's working in real conversations
- **Add new documents**: When new pitch decks or case studies are created, link them
- **Categorize consistently**: Use consistent category and domain names across projects
- **Link to services**: Every project should be linked to its parent service
